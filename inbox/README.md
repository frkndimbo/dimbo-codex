# Inbox State

One-line summary: store bulky or temporary working state here so the main instruction context stays clean.

## Rules
- One note per topic.
- Put a one-line summary at the top.
- Keep each note under 1,000 words.
- Add `status: active`, `status: stale`, or `status: resolved` near the top.
- Do not store secrets, tokens, private keys, raw auth logs, or full sensitive prompts.
- Delete or resolve transient notes in the same session when no longer needed.

## Naming
- Use `YYYY-MM-DD-short-topic.md`.
- Prefer stable names for ongoing work, such as `codex-config-audit.md`.
